기능 제거
항로 승인(채택) 워크플로
CR-SAM-UAM-001 / 003  ·  영향도 High
제거
확인된 사실
시스템 내부 승인 절차가 불필요하며, UAM 항로 승인 주체는 국토교통부. 현재 설계하는 시스템은 국토부와 독립적으로 운영되며 상호 연계 일정 미정.
제거되는 기능
시뮬레이션 결과 승인/반려 처리
PENDING → APPROVED → ACTIVE 상태 전환
운영항로 승격 처리 (APPROVED_DYNAMIC_ROUTE 생성)
항로 일정관리의 승인 워크플로 UI
대체 방향
시뮬레이션 = 단순 공간분석·경로평가 도구로 재정의
분석 결과는 참고 데이터로만 활용
국토부 연계 인터페이스는 미래 확장 포인트로 예약 (IF-SAM-008)
화면 레이블 변경: "승인" → "분석 완료", "채택" → "참고 경로 지정"
CI-UAM-2001 CI-UAM-2003 CI-UAM-3002 DB: APPROVED_*_ROUTE SFR-018 계열 DT-MENU-241~255
기능 변경
스케줄-항로 연동 방식
CR-SAM-UAM-002  ·  영향도 Medium
변경
확인된 사실
시뮬레이션에서 채택된 경로가 실제 운항경로로 사용될 가능성이 거의 없음. 그러나 스케줄과 항로의 조인 구조 자체는 향후 관리를 위해 유지 필요.
변경 내용
스케줄-항로 조인 구조 유지
FLIGHT_SCHEDULE에 실제 운항 적용 여부 플래그 추가 (RTE_APPLY_YN)
스케줄 등록 시 "실제 운항 적용" 선택 가능 (운영자 판단)
미적용 선택 시 사유 입력란 활성화
관제화면: RTE_APPLY_YN=Y 항로만 실선 표시
CI-UAM-3001 DB: FLIGHT_SCHEDULE DT-MENU-254
시뮬레이션 서비스 범위 재정의
CR-SAM-UAM-003  ·  영향도 High
변경
확인된 사실
UAM 항로는 국토부 승인 사항으로, 현 시스템의 승인 프로세스는 실제 행정 절차와 무관. 현 단계에서 국토부 시스템 연계는 범위 외.
변경 내용
시스템 내부 항로 "승인" 개념 제거
시뮬레이션 = 분석·검토·시각화 도구로 재정의
분석 상태값 재정의: PENDING/APPROVED → DRAFT/ANALYZED/ARCHIVED
국토부 연계 인터페이스 예약 등록 (IF-SAM-008, 시점 미정)
SFR-018 계열 전반 CI-UAM-2001~2003 요구사항 추적표
